üîé Step 1: Shifting Focus to ICMP Traffic

Upon closer inspection, I noticed that certain ICMP packets contained data that ended with what appeared to be Base64-encoded characters. This was definitely suspicious and worth investigating further.

üîç Step 2: Pattern Recognition

Diving deeper into the ICMP packets, I discovered a pattern. The packets that contained the suspicious data all shared common characteristics:

  • üéØ Source IP: 8.8.8.8
  • üÜî ICMP ID: 0x1337 (how fitting for a CTF!)
  • üì¶ They contained Raw payload data

I extracted the first packet matching these criteria and tested the ending characters with CyberChef. My suspicion was confirmed - they were indeed Base64-encoded data! üéâ

üíª Step 3: First Attempt and Revelation

My initial approach was to extract all packets with the identified characteristics and concatenate their payloads. However, when I decoded the result, the flag was garbled! üò§

That's when I had my "aha!" moment - I noticed the ICMP packets had sequence numbers, and they weren't arriving in order in the capture file!

üß© Critical Insight: The packets needed to be sorted by their ICMP sequence number before concatenating the data. The flag was split across multiple ICMP packets and needed to be reassembled in the correct order!

üêç Step 4: Script Development

I developed a Python script using Scapy to properly extract and reassemble the hidden message:

from scapy.all import rdpcap, IP, ICMP, Raw
import base64

# Read packets from the given pcap file
packets = rdpcap("unknown-traffic1.pcap")

# This will store the concatenated data from selected ICMP packets
d = b""

# Filter only packets that:
#  - have an IP layer with source address 8.8.8.8
#  - contain an ICMP layer
#  - contain a Raw payload
icmp_pkts = [pkt for pkt in packets if IP in pkt and pkt[IP].src == "8.8.8.8" and ICMP in pkt and Raw in pkt]

# Sort the selected packets by their ICMP sequence number
icmp_pkts.sort(key=lambda x: x[ICMP].seq)

# Iterate through the ordered packets
for pkt in icmp_pkts:
    data = pkt[Raw].load
    
    # Concatenate the payload to the buffer
    d += data

# Try to decode the concatenated data as Base64
try:
    decoded = base64.b64decode(d)
    # Print the decoded result as text (assuming it contains a flag or message)
    print("Flag:", decoded.decode())
except Exception as e:
    # If something goes wrong (invalid Base64, decoding error, etc.)
    print("Error:", e)

üß† How this works:

  1. Extract all ICMP packets from source 8.8.8.8 that contain raw data
  2. Sort these packets by their ICMP sequence number
  3. Concatenate all the payload data in the correct order
  4. Decode the resulting Base64 string to reveal the flag
